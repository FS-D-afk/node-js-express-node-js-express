# Changelog

本项目遵循语义化版本号。

## [1.2.2] - 2026-07-23

### Fixed

- 修复订单到期后数据库仍返回旧 `pending` 订单，导致无法重新下单的问题。
- 修复 SQLite UTC 时间与 Node.js 北京时间混用造成的 8 小时时差。
- 修复旧订单上传截图后提示过期，但仍占用有效订单唯一索引的问题。
- 修复订单状态轮询未及时执行过期清理的问题。

### Added

- 历史 510 分钟异常订单自动迁移。
- 过期和取消订单的“重新下单”入口。
- `/healthz` 健康检查。
- PM2 配置、Ubuntu/Nginx 部署文档和升级回滚文档。
- GitHub Actions 语法检查与回归测试。
- `ADMIN_USERNAME`、`ADMIN_PASSWORD` 首次初始化配置。
- `npm run check` 综合检查命令。

### Security

- 生产环境要求至少 32 个字符的 `SESSION_SECRET`。
- 生产环境新建数据库时要求至少 8 位的管理员初始密码。
- 管理员后台修改密码的最低长度提高为 8 位。
- GitHub 发布包排除真实 `.env`、数据库、用户上传、日志和 Git 历史。
