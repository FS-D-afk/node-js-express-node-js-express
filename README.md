# 期末资料自动售卖系统 v1.2.1

这是一个低成本的数字资料自助售卖系统。前台用户必须使用邮箱和密码登录，订单永久绑定账号；同一商品对所有用户显示并收取相同价格，不再通过增加 0.01～0.89 元尾数区分订单。

## 本版主要功能

- 强制邮箱 + 密码登录，不接短信，也不发送日常验证邮件。
- 登录状态默认保持 30 天，服务器重启后仍有效。
- 新订单直接绑定用户账号，支持跨设备查看“我的订单”。
- 同一账号购买同一商品时，只保留一个有效订单；重复点击不会生成重复订单。
- 所有用户按商品后台标价付款，`pay_amount` 与商品价格一致。
- 自动发货只校验截图中识别出的付款金额是否与订单金额一致。
- 不再要求截图必须包含“支付成功”文字或交易单号，也不再阻止相同截图重复提交。
- 截图格式或金额识别失败时，显示必须点击“确定”才能关闭的弹窗。
- 已支付资料永久不限次数读取。
- 后台用户管理支持按邮箱搜索和管理员重置密码；重置后该用户所有设备自动退出。
- 保留原版商品、订单、收款码、上传文件、公告及 SQLite 数据，并在启动时自动迁移数据库。

## Windows 启动

在 CMD 中进入项目目录：

```bat
cd /d "你的项目完整路径"
copy .env.example .env
npm install
npm start
```

浏览器打开：

```text
http://localhost:3000
```

系统会先显示用户登录页。默认后台地址和账号：

```text
地址：http://localhost:3000/admin
账号：admin
密码：admin123
```

首次部署后请立即修改后台密码，并把 `.env` 中的 `SESSION_SECRET` 改成长随机字符串。

## 用户购买流程

1. 用户注册或登录邮箱账号。
2. 进入商品页面并点击“立即购买”。
3. 系统创建账号订单；重复点击会返回原订单。
4. 用户按商品标价付款，不需要支付额外尾数。
5. 上传能清楚显示付款金额的截图；付款完成页或账单详情页均可。
6. OCR 识别金额与订单金额一致后，系统自动发货。
7. 金额未识别或不一致的订单，可重新上传或由管理员人工确认。

例如订单金额为 `￥40.00`，截图中能够识别出 `40.00` 即可通过，不要求出现交易单号或“支付成功”文字。

## 数据库自动升级

首次启动会自动创建或补充：

- `users`：用户邮箱和加密密码。
- `user_sessions`：长期登录令牌摘要。
- `orders.user_id`：订单所属用户。
- `orders.customer_token`：兼容原浏览器订单。
- `payment_proofs`：保存截图、OCR 原文和识别金额。
- 数据库唯一约束：同账号同商品仅一个待处理订单。

旧版的交易单号字段和索引会保留以兼容历史数据，但新上传截图不再使用交易单号进行判断。

无需手工执行 SQL。升级前仍建议备份：

```text
data/app.db
public/uploads
```

## 管理员重置用户密码

进入：

```text
后台首页 → 用户
```

管理员可搜索邮箱并设置新密码。密码重置后，该用户全部登录令牌立即失效。由于本方案不接短信和邮件服务，管理员需要通过现有售后渠道把新密码告知用户。

## OCR 配置

Windows 默认调用项目内的 `scripts/windows-ocr.ps1`。也可以在 `.env` 中配置其他 OCR：

```text
OCR_COMMAND=tesseract {file} stdout
```

支持 PNG、JPG、WEBP 和 BMP，单张最大 5MB。为了提高识别率，截图中的付款金额应清晰、无遮挡，建议保留两位小数，例如 `40.00`。

## 风险说明

金额校验是最宽松、成本最低的自动确认方式，但同价订单之间无法仅凭金额证明付款属于哪个用户。账号绑定只能限制用户访问自己的订单，不能替代微信或支付宝官方支付回调。发生争议时需通过后台订单和付款记录人工复核。
