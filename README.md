# 期末资料售卖系统 v1.3.1

这是一个低成本的数字资料售卖系统。前台用户使用邮箱和密码登录，订单绑定账号；付款截图只用于辅助人工审核，不能替代微信、支付宝或其他支付平台的服务端回调。

## 本版主要功能

- 强制邮箱 + 密码登录，不接短信，也不发送日常验证邮件。
- 用户新密码至少 12 位；修改或管理员重置密码后，其他设备自动退出。
- 登录状态默认保持 30 天，服务器重启后仍有效。
- 普通用户登录同时按“客户端地址 + 邮箱”和客户端地址总量限流。
- 同一客户端地址每小时最多尝试注册 5 次。
- 不存在的用户或管理员账号仍执行密码哈希比较，降低通过响应时间枚举账号的风险。
- 新订单直接绑定用户账号，支持跨设备查看“我的订单”。
- 同一账号购买同一商品时，只保留一个有效订单；重复点击不会生成重复订单。
- 所有用户按商品后台标价付款，`pay_amount` 与商品价格一致。
- 上传付款截图后统一进入人工审核，不再根据 OCR 金额自动发货。
- OCR 结果、识别金额、交易单号和截图复用情况仅作为管理员审核参考。
- 相同截图用于其他订单时，后台显示重点审核警告。
- 已支付资料永久不限次数读取。
- 后台用户管理支持按邮箱搜索和管理员重置密码；重置后该用户所有设备自动退出。
- 生产环境必须使用强 `SESSION_SECRET`，首次启动必须显式配置管理员账号和密码。
- 所有状态变更请求启用基于 Session 的 CSRF 校验。
- 后台登录失败按客户端地址限流：15 分钟内最多 5 次。
- 启用 CSP、防点击劫持、禁止 MIME 猜测、敏感页面禁止缓存等浏览器安全响应头。

## Windows 启动

在 CMD 中进入项目目录：

```bat
cd /d "你的项目完整路径"
copy .env.example .env
npm install
npm start
```

浏览器打开：

```text
http://localhost:3000
```

首次启动前必须编辑 `.env`：

```dotenv
NODE_ENV=development
SESSION_SECRET=至少32位的随机字符串
ADMIN_USERNAME=你的管理员用户名
ADMIN_PASSWORD=至少12位的强密码
TRUST_PROXY=0
```

项目不再创建 `admin/admin123` 默认账号。管理员表为空且缺少上述配置时，应用会拒绝启动。

正式部署时必须设置：

```dotenv
NODE_ENV=production
```

生产环境下，缺失、过短或仍使用示例值的 `SESSION_SECRET` 会导致启动失败。

`TRUST_PROXY` 默认必须保持为 `0`。只有在应用确实位于一层可信反向代理（例如由自己控制的 Nginx）之后时，才设置为 `1`；否则攻击者可能伪造转发地址，绕过基于客户端地址的登录限流。

## 用户购买流程

1. 用户注册或登录邮箱账号。
2. 进入商品页面并点击“立即购买”。
3. 系统创建账号订单；重复点击会返回原订单。
4. 用户按商品标价付款。
5. 用户上传付款截图。
6. 系统保存截图、图片哈希、OCR 原文、识别金额和可识别的交易单号。
7. 订单进入“待审核”，由管理员结合真实收款记录人工确认。
8. 管理员确认付款后，系统才开放资料发货页。

即使 OCR 识别到与订单相同的金额，也不会自动标记已支付。金额相同无法证明付款属于特定订单。

## 数据库自动升级

首次启动会自动创建或补充：

- `users`：用户邮箱和加密密码。
- `user_sessions`：长期登录令牌摘要。
- `orders.user_id`：订单所属用户。
- `orders.customer_token`：兼容原浏览器订单。
- `payment_proofs`：保存截图、图片哈希、OCR 原文、识别金额、交易单号和审核原因。
- 数据库唯一约束：同账号同商品仅一个待处理订单。

交易单号不再设置唯一约束，因为同一张截图或同一交易信息可能被重复提交；系统会保留记录并在后台提示人工复核。

升级前建议备份：

```text
data/app.db
public/uploads
```

## 管理员重置用户密码

进入：

```text
后台首页 → 用户
```

管理员可搜索邮箱并设置至少 12 位的新密码。密码重置后，该用户全部登录令牌立即失效。由于本方案不接短信和邮件服务，管理员需要通过现有售后渠道把新密码告知用户。

## OCR 配置

Windows 默认调用项目内的 `scripts/windows-ocr.ps1`。也可以在 `.env` 中配置其他 OCR：

```text
OCR_COMMAND=tesseract {file} stdout
```

支持 PNG、JPG、WEBP 和 BMP，单张最大 5MB。OCR 仅用于减少人工录入，不具备付款真实性校验能力。

## 安全说明

- `.env`、SQLite 数据库、上传截图和 `node_modules` 不应提交到 Git。
- 截图金额、支付成功文字和交易单号都不能代替支付平台服务端回调。
- 管理员确认订单前，应在真实收款渠道中核对付款记录。
- 当前 CSRF 令牌只在请求体或同源请求头中提交，不写入 URL。
- 当前登录与注册限流保存在单进程内存中；多实例部署应更换为 Redis 等共享存储。
- 注册流程目前只校验邮箱格式，不验证邮箱归属。
- 正式公开运营前仍需补充邮件验证、找回密码流程和支付平台服务端回调。
