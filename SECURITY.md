# Security Policy

## 敏感数据

以下内容不得提交到公开仓库：

- `.env`
- SQLite 数据库及 `-wal`、`-shm` 文件
- 付款截图和用户上传图片
- 日志、备份和服务器配置
- 私钥、证书、访问令牌和第三方 API 密钥

`.gitignore` 已包含常见排除规则，但提交前仍应检查：

```bash
git status
git diff --cached
```

## 生产配置

- `SESSION_SECRET` 至少 32 个随机字符。
- 管理员密码和用户密码不得复用。
- 使用 HTTPS；生产 Cookie 依赖 HTTPS。
- 限制 `.env` 权限：`chmod 600 .env`。
- 数据库和上传目录应只允许服务账号读取。
- 定期备份，并加密保存包含用户数据的备份。

## 已经泄露的凭据

若 `.env`、数据库或 Token 曾出现在公开 GitHub 历史中，应视为已经泄露。仅删除当前分支中的文件不足以恢复安全，必须立即轮换：

- `SESSION_SECRET`
- 管理员密码
- GitHub Personal Access Token
- 第三方 API 密钥
- 数据库账号密码（外部数据库场景）

更换 `SESSION_SECRET` 会使现有后台会话失效，这是正常现象。

## 支付风险

OCR 金额匹配不等同于支付平台官方回调。争议订单、同价订单和异常截图需要管理员结合真实收款记录复核。

## 报告漏洞

请使用 GitHub 仓库的私有安全报告功能或联系仓库维护者。报告中不要附带真实用户数据库、支付截图或有效密钥。
